# Yoyoo Web 验收清单（V1）

本清单用于发布前确认“前端 + 后端真实链路”可用，避免回归到预制回复或 mock 任务中心。

## 前置条件
- 后端已启动（示例）：`http://127.0.0.1:18000`
- 前端已启动（示例）：`http://127.0.0.1:3000`
- 前端环境变量：
  - `YOYOO_BACKEND_BASE_URL=http://127.0.0.1:18000`

## 一键验收
```bash
cd YoyooTest0.2
npm run acceptance:v1
```

通过标准：输出 `验收完成：6/6 通过`。

## 6 个核心场景
1. 普通对话：CEO 正常回复（非报错/非空）。
2. 任务触发：任务类请求进入“确认执行”流程。
3. 长任务进度：确认后返回 `task_id`，任务中心可见并有时间线。
4. 失败重试：消息重复提交幂等去重（同 `dedupeKey` 仅写入一次）。
5. 刷新历史：服务端历史持久化，刷新后可重载。
6. 多会话切换：会话 A/B 数据隔离，不串线。

## 相关接口
- `POST /api/chat/stream`
- `GET/POST /api/chat/messages`
- `GET /api/chat/tasks`
- 后端：`/api/v1/team/*`
