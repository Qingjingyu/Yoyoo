# AI CEO-Yoyoo Mode v1 总纲（冻结版）

> 日期：2026-02-09  
> 状态：Frozen v1（先定模式，暂不扩展功能）

## 1. 产品定义

Yoyoo 是唯一对外入口与唯一大脑（AI CEO），不是执行工具本身。  
用户只与 Yoyoo 沟通，Yoyoo 负责理解目标、规划任务、调度执行、质量验收、记忆沉淀。

一句话目标：把“聊天式助手”升级为“可持续协作的任务型大脑”，让任务可追踪、可恢复、可迭代。

## 2. 组织与协作模型

### 2.1 组织结构

- `Yoyoo`：CEO（唯一大脑、唯一对外负责人）
- `OpenClaw`：执行经理 A（主执行路径）
- `Nanobot`：执行经理 B（备份/并行路径）
- 其他工具（如 OpenCode/Codex）：员工（按需被经理调度）

### 2.2 指挥链

标准链路：`用户 -> Yoyoo -> 执行经理 -> 工具员工 -> Yoyoo -> 用户`

执行规则：
- 用户只与 Yoyoo 对话。
- 执行层不直接对用户做最终承诺。
- 所有结果必须回流 Yoyoo 统一验收与回复。

### 2.3 职责分工

Yoyoo（CEO）负责：
- 需求澄清与目标定义
- 任务拆解与优先级管理
- 统一记忆归档与策略沉淀
- 结果质量判定与复盘

执行经理（Claw/Nano）负责：
- 承接子任务并执行
- 编排下属工具
- 提交执行证据（日志、输出、状态）
- 报告失败原因与阻塞点

## 3. 记忆模型（联邦记忆）

- Yoyoo 保存“公司总账记忆”（权威版本）。
- 执行经理保存“本地工作记忆”（短期执行上下文）。
- 同步策略：本地先记，异步回传，最终由 Yoyoo 归档。
- 冲突规则：Yoyoo 版本为准，同时保留原始冲突快照用于审计。

## 4. 能力接入与升级治理

### 4.1 能力分类（先分类再接入）

- `Brain 能力`：规划、记忆、策略、验收、复盘
- `Executor 能力`：代码执行、命令执行、浏览器操作、部署
- `Channel 能力`：飞书/钉钉/微信接入
- `Ops 能力`：监控、告警、自愈、审计

分类原则：
- “思考与记忆”优先进 Brain。
- “动作执行”优先进执行层。

### 4.2 标准接入流程（固定 6 步）

1. 提案：收益、成本、风险、回滚方案。  
2. 适配：先接 Adapter，不允许直连 Brain 内核。  
3. 沙箱评测：统一任务集对比成功率、时延、稳定性。  
4. 灰度：小流量上线并监控。  
5. 晋升：达标后进入生产主路径。  
6. 复盘：沉淀策略卡，更新禁用/推荐清单。  

### 4.3 准入门槛

- 有明确收益（成功率、速度或稳定性至少一项显著提升）。
- 支持开关控制与快速回滚。
- 可观测（日志、错误分类、健康状态）。

不满足任一条，禁止上线主路径。

### 4.4 复杂度控制

- 生产默认一主一备：Claw 主、Nano 备。
- 每两周最多引入一个新能力到灰度。
- 引入新主能力时必须定义旧能力退场路径。

## 5. v1 非目标

- 不追求一次接入大量执行器。
- 不做复杂自治组织与全自动跨团队调度。
- 不在 v1 引入无法回滚的新基础设施。

## 6. v1 成功指标

- 每个任务都有 `task_id`。
- 支持“继续上次任务”。
- 每次执行结果有通过/不通过判定。
- 重启后可恢复最近任务上下文。

## 7. 对外承诺

- 你只需要找 Yoyoo，不需要分别协调执行器。
- 失败时必须给出原因、证据和下一步建议。
- 所有关键决策都可追溯到任务记录。
