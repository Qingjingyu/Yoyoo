# Yoyoo 员工激活基座 v1

## 目标
把 Yoyoo1.0 做成可复制的员工基座：任何新员工从 Git 拉取后，激活即具备统一初始能力。

## 默认初始能力（激活即有）
- OpenClaw latest
- MiniMax 接入配置
- QMD 记忆后端
- 基础技能包（7 个）
- 角色身份模板（CEO/后勤/研发总监/研发员工）
- 网关守护与健康检查

## 激活方式
1. 服务器准备：Ubuntu + root 权限。
2. 设置 `MINIMAX_API_KEY`。
3. 运行：
   - `bash Yoyoo/project/bootstrap/activate_employee.sh`
   - 或 `bash Yoyoo/project/bootstrap/hire_employee_from_git.sh`

## 组织落地规则
- 新员工必须从本基座激活，禁止手工漂移配置。
- 公共能力进入 baseline 前，必须先在沙箱验证。
- 需要额外 key 的能力，默认归类到 optional。

## 维护机制
- 基线能力清单：`Yoyoo/project/bootstrap/registry/capability-catalog.yaml`
- 角色模板：`Yoyoo/project/bootstrap/profiles/`
- 激活入口：`Yoyoo/project/bootstrap/activate_employee.sh`
