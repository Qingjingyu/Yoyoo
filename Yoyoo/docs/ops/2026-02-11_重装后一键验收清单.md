# 重装后一键验收清单（5 条命令）

用途：服务器重装并重新接入后，用 2 分钟确认 `CEO/后勤 + MiniMax + 飞书` 全链路可用。

## 1) CEO 网关健康
```bash
openclaw --profile ceo gateway status --deep
```
期望：`Runtime: running` 且 `RPC probe: ok`。

## 2) 后勤网关健康
```bash
openclaw --profile ops gateway status --deep
```
期望：`Runtime: running` 且 `RPC probe: ok`。

## 3) CEO 飞书通道探活
```bash
openclaw --profile ceo channels status --json --probe
```
期望：`configured=true`、`running=true`、`probe.ok=true`。

## 4) 后勤飞书通道探活
```bash
openclaw --profile ops channels status --json --probe
```
期望：`configured=true`、`running=true`、`probe.ok=true`。

## 5) 双实例模型回包（MiniMax）
```bash
openclaw --profile ceo agent --to clawceo --message "只回复 pong" --json && \
openclaw --profile ops agent --to clawops --message "只回复 pong" --json
```
期望：两边 `payloads[0].text` 都返回 `pong`，且无 `401`。

## 失败即处理（简版）
- 若第 1/2 条失败：先 `openclaw --profile <ceo|ops> gateway restart`，再重测。
- 若第 3/4 条失败：查 `openclaw --profile <ceo|ops> channels logs`。
- 若第 5 条失败且报 `401`：按 `2026-02-10_OpenClaw_MiniMax_快速接入与401排障SOP.md` 处理。
