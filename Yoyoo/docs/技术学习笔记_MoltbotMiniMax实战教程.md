## 2. Moltbot 核心特点

### 2.1 定位

| 特性 | 说明 |
|------|------|
| **7x24 小时运行** | 全天候 AI 助理 |
| **本地存储** | 数据存在本地，保护隐私 |
| **持久记忆** | 记得几周前随口提过的事 |

```
┌─────────────────────────────────────────────────────────┐
│              传统 AI vs Moltbot                         │
├─────────────────────────────────────────────────────────┤

│  传统 AI:                                               │
│  "人找 AI" - 用户发起对话，AI 响应                      │
```

## 4. 安装部署

### 4.1 一键安装

```bash
curl -fsSL https://molt.bot/install.sh | bash
```

### 4.2 安装流程

```
1. 执行安装命令
   ↓
2. 确认风险提示 → 选择 "Yes"
   ↓
3. 选择初始化模式 → "Quick Start"
   ↓
4. 选择模型 → MiniMax M2.1
   ↓
5. 输入 API Key (从 MiniMax 官网获取)
   ↓
6. 选择默认模型和可用 app
   ↓
7. 配置 IM (Telegram / 飞书)
   ↓
8. 安装技能 (可选，建议"Yes")
   ↓
9. 配置 npm 包管理器
   ↓
10. 选择要安装的 Skills
    ↓
11. 配置 hooks (建议全开)
    ↓
12. 启动服务
    ↓
13. 验证端口 (18789)
```

### 4.3 验证服务

```bash
# 检查端口是否监听
ss -lntp | grep 18789

# 输出示例:
# LISTEN  0  511  *:18789  *:*  users:(("clawdbot",pid=1234,fd=20))

# 如果没有输出，需要配置 systemd service
```

### 4.4 启动服务

```bash
# 启动网关 (verbose 模式)
clawdbot gateway --verbose

# 默认端口: 18789
```

## 5. Telegram 配置

### 5.1 创建 Bot

```
1. 打开 Telegram，搜索 @botfather
2. 发送 /newbot
3. 设置机器人名称
4. 复制 Bot Token
5. 在终端输入 Token
```

### 5.2 配对

```
1. 在 Telegram 中和机器人对话 /start
2. 获取配对码
3. 在服务器终端执行:
   clawdbot pairing approve telegram <配对码>
```

## 6. 飞书配置

### 6.1 飞书开放平台

```
1. 访问 open.feishu.cn
2. 创建应用
3. 记录 App ID 和 App Secret
```

### 6.2 安装飞书插件

```bash
# 在 Moltbot 小龙虾界面输入
clawdbot plugins install @m1heng-clawd/feishu
```

### 6.3 配置步骤

```
1. 发送 App ID 给 Moltbot
2. 发送 App Secret 给 Moltbot
3. 配置相关权限
4. 在飞书中添加机器人
5. 开通相关权限
6. 配置事件与回调
7. 选择长连接模式
```

### 6.4 飞书插件仓库

```
GitHub: https://github.com/m1heng/Clawdbot-feishu
```

## 11. 学习总结

### 核心要点

1. **Clawdbot 已更名为 Moltbot** - 商标问题
2. **M2.1 是最佳搭档** - 速度快、稳定、不啰嗦
3. **一键安装** - `curl -fsSL https://molt.bot/install.sh | bash`
4. **默认端口 18789** - WebSocket 端口
5. **飞书集成成熟** - 有官方插件
6. **7x24h 稳定运行** - 心跳机制

### 对 Yoyoo 的价值

- Body 方案：直接使用 Moltbot
- IM 接入：参考飞书插件
- 模型选择：考虑 M2.1 (成本 + 性能)
