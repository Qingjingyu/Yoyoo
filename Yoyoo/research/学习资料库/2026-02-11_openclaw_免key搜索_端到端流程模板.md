# OpenClaw 免 Key 搜索端到端流程模板

- 日期：2026-02-11
- 适用：当前服务器默认路线（`duckduckgo-search` + Browser）

## 1. 标准任务链路

1. 搜索：先用 `duckduckgo-search` 找 3-5 条候选链接  
2. 打开：用 Browser 打开前 1-2 条链接  
3. 提炼：抽取要点、证据、时间、结论  
4. 输出：给用户“结论 + 证据链接 + 下一步建议”

## 2. 给 CEO 的固定提示词（可直接发）

```text
按以下流程执行并只输出最终结果：
1) 用 duckduckgo-search 搜索：<你的问题>
2) 选最相关的 2 个结果，用浏览器打开并读取正文
3) 输出：
   - 结论（3条以内）
   - 证据（链接+一句话）
   - 风险/不确定性（如有）
   - 下一步建议（1-2条）
要求：不要编造，没查到就明确说没查到。
```

## 3. 验收标准

1. 输出中必须有可点击链接  
2. 结论必须能被链接内容支撑  
3. 有时间敏感信息时，必须写明日期  
4. 查不到时必须显式说明“未检索到充分证据”

## 4. 失败时的降级策略

1. 浏览器失败：先只返回搜索结果与来源  
2. 搜索失败：提示用户改关键词并给 3 个可替代关键词  
3. 结果冲突：并列给出两种说法并标注来源

## 5. 说明

当前已下线 `brave-search-setup` 与旧 `ddg-search`，默认只走：
- `duckduckgo-search`（免 Key 搜索）
- `playwright-browser-automation`（网页读取/操作）
