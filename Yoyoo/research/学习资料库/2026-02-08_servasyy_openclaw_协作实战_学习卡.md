# 学习卡：OpenClaw 多 Agent 协作实战（servasyy_ai）

- 来源：`https://x.com/servasyy_ai/status/2020475413055885385`
- 抓取时间：2026-02-08
- 适用目标：构建 Yoyoo「长期记忆 + 可进化 + 人在环」的大脑体系

## 一句话结论
这份材料的核心价值不在“多 Agent 数量”，而在“文件化记忆 + 人格契约 + 人在环 + 可插拔技能”的长期协作方法。

## 可复用要点（给 Yoyoo）
1. 记忆分层：长期记忆（稳定事实/决策）+ 日志记忆（当天过程/反思）。
2. 人格契约：每个角色有明确价值观、边界和输出风格，避免角色漂移。
3. 人在环：重要动作可审批、可回滚，用户始终保留最终决策权。
4. Skills 生态：能力模块化、可插拔，不把所有能力硬塞进主程序。
5. 会话路由：多角色协作要靠“会话隔离 + 明确消息转发协议”，而不是群里自由聊天。

## 对我们当前问题的映射
1. 你之前觉得“两个机器人不清醒”，本质是缺少统一主持与协作协议。
2. 这篇文章支持我们现在的方向：
   - 对外只保留一个 Yoyoo 身份
   - 内部通过会话路由调度执行体
   - 记忆和决策统一收口到 Yoyoo Brain

## 落地到 Yoyoo Kernel 1.0 的动作
1. 建立 `SOUL.md`（人格契约）模板：角色、边界、原则、输出格式。
2. 建立 `MEMORY.md` + `memory/daily/` 双层记忆结构。
3. 建立“审批闸门”：高风险动作必须 `confirm -> execute -> evidence -> close`。
4. 建立内部协作协议：`task_id, owner, status, evidence, next_action`。
5. 建立 skills 入库流程：`导入 -> 沙箱验证 -> 评分 -> 上线/回滚`。

## 采用策略
- 采用：方法论与协作原则。
- 不直接照搬：具体人设文案、社区叙事、平台绑定细节。
- 我们的版本：以 OpenClaw 为执行底盘，Yoyoo Brain 统一决策与记忆。
