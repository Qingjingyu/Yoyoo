# memU Bot 体验与对比

> **调研日期**: 2026-02-05
> **来源**: 用户长文体验分享
> **目标**: 记录 memU Bot 的部署体验、能力取向，与 Clawdbot 的对比。

## 1. 概览
- 轻量化 7×24 Agent：客户端安装 + 填模型 API Key + 接入通讯工具（飞书等），几分钟上手。
- 核心取向：**长期运行 + 记忆为底座**，能力按需装（Skills/MCP），不内置一堆冗余能力。
- 体验卖点：后台持续跑任务，不需反复确认；记住用户偏好，输出更贴合个人视角。

## 2. 部署体验
- 下载地址：https://memu.bot/
- 部署路径：安装客户端 → 配置模型 API Key → 绑定通讯工具（飞书）→ 即可呼叫。
- 官方教程（飞书示例）：https://memu.bot/tutorial/feishu
- 相比 Clawdbot/Moltbot：无环境/依赖折腾，面向普通用户；但能力通过 Skills/MCP 扩展。

## 3. 使用表现
- 记忆驱动：能记住使用习惯，自动避开“每次从零开始问格式”的窘境。
- 后台持续：对话不中断也继续跑（下载/转录/生成文档、PPT、摘要等）。
- 任务示例：
  - 链接阅读 → 生成贴合个人视角的摘要
  - 自动产出 Word/PPT 并告知存储位置
  - B 站视频链接 → 后台下载 mp4 + 提取文字 + 封面/结构化 json
- 自定义提示：可设定称呼/人格化提示（如称呼“海总”）。

## 4. Skills & MCP
- 安装/列举/查找技能均可对话式完成。
- 思路：**按需装、能用就装，不用不装**，避免默认堆砌。
- MCP 同理：按需加载外部服务，任务里调用。

## 5. 与 Clawdbot/Moltbot 对比
- Clawdbot：工程复杂度高、可玩性强，但部署门槛高，更像“能力集合”。
- memU Bot：产品化更强，部署丝滑，更适合“7×24 代办”落地；扩展靠 Skills/MCP。

## 6. 对 Yoyoo 的启示
- 作为轻量常驻助手的可选方案：部署成本低，适合非工程背景的用户。
- 需评估：可扩展性（自定义 Skills/MCP）、安全与权限、成本（模型 API）。
- 可与我们三层记忆/SOP 结合：导入记忆、个性化提示，保持任务持续性。
