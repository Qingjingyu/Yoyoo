## 2. 核心洞察

### 2.1 OpenClaw 的定位

> **"中层管理"** - Peter（OpenClaw 创作者）

```
┌─────────────────────────────────────────────────────────┐
│                     你的角色                            │
│         建筑师（设计结构）/ 大老板（指明方向）           │
└─────────────────────┬───────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────┐
│                   OpenClaw                              │
│         中层管理 - 拆解任务、分配任务、汇报               │
└─────────────────────┬───────────────────────────────────┘
                      │
          ┌───────────┼───────────┐
          ▼           ▼           ▼
     ┌────────┐  ┌────────┐  ┌────────┐
     │ AI员工  │  │ AI员工  │  │ AI员工  │
     └────────┘  └────────┘  └────────┘
```

### 2.2 效率跃迁

| 阶段 | 方式 | 效率 |
|------|------|------|
| 1 | 人写代码 | 基准 |
| 2 | 人指挥 AI 写代码 | 10x |
| 3 | 人指挥 AI 管理 AI 写代码 | 100x+ |

**Peter 的数据**: 一天提交 600 个 commit，自己都没看过

### 2.3 关键认知

> "软件开发中真正难的是想清楚要做什么、怎么做、为什么这样做。代码只是把想法翻译成机器能理解的语言。"

**建议**: 花更多时间在 **thinking** 上，而不是 **typing** 上。

## 3. 安装与升级指南

### 3.1 安装命令

```bash
# macOS / Linux
curl -fsSL https://openclaw.ai/install.sh | bash

# Windows
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### 3.2 升级注意事项

**问题**: 从 Clawdbot/Moltbot 升级时，文件夹改名会导致配置不适配

**解决方案**:
```bash
# 1. 备份配置文件
cp -r ~/.clawdbot ~/.clawdbot.backup
cp -r ~/clawd ~/clawd.backup

# 2. 运行更新脚本
curl -fsSL https://openclaw.ai/install.sh | bash

# 3. 出现 "是否删除旧数据" 时，选择 **不要删除**
```

### 3.3 macOS 26 注意事项

```bash
# 必须先安装 Xcode 命令行工具
xcode-select --install

# 否则 Skills 安装会报错
```

## 5. 卸载清理

### 5.1 完全卸载 OpenClaw

```bash
openclaw uninstall
# 选择全部删除，包括客户端 App
```

### 5.2 清理残留 Clawdbot

```bash
clawdbot uninstall

# 如果 CLI 残留，手动清理
npm rm -g clawdbot
npm rm -g openclaw
```

## 8. Peter 的工作方式

### 8.1 代码验证机制

```
AI 写代码 + AI 写测试
       ↓
   测试通过
       ↓
    敢发布
```

### 8.2 PR Review 方式

**传统方式**: 看代码本身

**Peter 的方式**: 看提交时用的 prompt

> "prompt 能告诉他这个人是怎么思考这个问题的，比看最终的代码更有价值。"

**建议**: PR 以后应该改名叫 **Prompt Request**
